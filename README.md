# Старт у залі — план тренувань (PWA)

Персональний додаток-тренування: фото тренажерів, відео техніки, таймер відпочинку і
**збереження прогресу** (галочки, робочі ваги, історія тренувань) прямо у браузері.

## Що це
Один статичний сайт без сервера. Працює офлайн і встановлюється на телефон як застосунок.

## Як викласти на GitHub Pages (5 хв)

1. Створи на GitHub новий **публічний** репозиторій, напр. `gym-start`.
2. Завантаж у нього всі файли з цієї теки:
   - `index.html`
   - `manifest.webmanifest`
   - `service-worker.js`
   - `icon-192.png`
   - `icon-512.png`
   (через кнопку **Add file → Upload files** на сторінці репозиторію).
3. Відкрий **Settings → Pages**.
4. У розділі *Build and deployment* → *Source* обери **Deploy from a branch**,
   гілка **main**, тека **/(root)**, натисни **Save**.
5. Зачекай ~1 хвилину. Зверху з'явиться адреса виду
   `https://ТВІЙ-НІК.github.io/gym-start/` — це і є твій додаток.

## Як встановити на телефон
- Відкрий адресу в браузері телефона.
- **Android (Chrome):** меню ⋮ → «Додати на головний екран» (або кнопка «Встановити»).
- **iPhone (Safari):** кнопка «Поділитися» → «На початковий екран».
- Після цього він відкривається як звичайний застосунок, працює офлайн.

## Прогрес і перенесення між пристроями
- Прогрес зберігається **на кожному пристрої окремо** (це особливість безкоштовного хостингу без сервера).
- Щоб перенести: вкладка **Прогрес → Експорт** на одному пристрої, потім **Імпорт** на іншому.

## Хочеш авто-синхронізацію між усіма пристроями?
Потрібен невеликий бекенд. Найпростіші безкоштовні варіанти:
- **Supabase** (Postgres + auth, безкоштовний тариф) — додаємо вхід і таблицю прогресу;
- приватний **GitHub Gist + токен** — без окремого сервера, синхронізація через GitHub API.
Скажи — підготую покрокову версію.
