# Старт у залі — план тренувань (PWA з синхронізацією)

Додаток-тренування: фото тренажерів, відео техніки, таймер відпочинку, історія тренувань
і **синхронізація прогресу між усіма твоїми пристроями** (телефон + комп'ютер).

Працює як звичайний застосунок на телефоні й офлайн. Прогрес завжди зберігається на пристрої,
а коли ти увійдеш — ще й однаковий скрізь.

---

## Частина 1. Викласти додаток на GitHub Pages (≈5 хв)

1. Зайди на **github.com**, увійди (або зареєструйся — безкоштовно).
2. Натисни **+** угорі праворуч → **New repository**.
3. Назва, напр. `gym-start`. Постав **Public**. Натисни **Create repository**.
4. На сторінці репозиторію: **Add file → Upload files**.
5. Перетягни **усі файли з цієї теки** (index.html, config.js, manifest.webmanifest,
   service-worker.js, icon-192.png, icon-512.png). Натисни **Commit changes**.
6. Зверху відкрий **Settings** → ліворуч **Pages**.
7. *Build and deployment* → *Source*: **Deploy from a branch**.
   Гілка **main**, тека **/(root)** → **Save**.
8. Зачекай ~1 хв, онови сторінку. З'явиться адреса виду
   **https://ТВІЙ-НІК.github.io/gym-start/** — це твій додаток. Відкрий, перевір.

Поки що працює локально (без синхронізації) — це нормально. Вмикаємо синхронізацію далі.

---

## Частина 2. Безкоштовна синхронізація через Supabase (≈10 хв)

Supabase — це безкоштовна «база даних у хмарі». Вона зберігатиме твій прогрес,
щоб він був однаковий на всіх пристроях. Сервер писати не треба.

### Крок 1. Створити проєкт
1. Зайди на **supabase.com** → **Start your project** → увійди (найшвидше — кнопкою **GitHub**).
2. **New project**.
3. *Name*: `gym-start`. *Database Password*: придумай і **збережи десь** (далі не знадобиться, але хай буде).
4. *Region*: обери ближчий — **Central EU (Frankfurt)**.
5. **Create new project**. Зачекай ~2 хв, поки створиться.

### Крок 2. Створити таблицю для прогресу
1. Ліворуч у меню вибери **SQL Editor** → **New query**.
2. Відкрий файл **supabase_setup.sql** з цієї теки, скопіюй увесь його текст
   і встав у вікно запиту.
3. Натисни **Run** (або Ctrl/Cmd+Enter). Має написати *Success*. Готово — таблиця і захист створені.

### Крок 3. Дозволити вхід поштою без підтвердження листа
(Щоб можна було увійти одразу, без очікування листа.)
1. Ліворуч **Authentication** → **Sign In / Providers** (в деяких версіях — **Providers**).
2. Знайди **Email** — він має бути увімкнений (зелений).
3. Відкрий його налаштування і **вимкни** опцію **Confirm email**
   (буває в Authentication → **Sign In / Providers → Email**, або в **Authentication → Settings**).
   Збережи. *Якщо не знайдеш — нічого, просто після реєстрації підтвердиш пошту листом один раз.*

### Крок 4. Скопіювати два ключі
1. Ліворуч натисни шестерню **Project Settings** → **API**.
2. Знадобляться два значення:
   - **Project URL** (виглядає як `https://abcd1234.supabase.co`)
   - **anon public** — довгий ключ у блоці *Project API keys*.
   > Цей ключ можна тримати у відкритому коді — він безпечний (доступ захищає правило з Кроку 2).

### Крок 5. Вставити ключі у додаток
1. Відкрий файл **config.js** (можна прямо на GitHub: натисни на файл → олівець ✏️ *Edit*).
2. Встав свої значення між лапками:
   ```js
   window.SUPABASE_URL = "https://abcd1234.supabase.co";
   window.SUPABASE_ANON_KEY = "встав_сюди_anon_public_ключ";
   ```
3. Збережи (**Commit changes**). За хвилину GitHub оновить сайт.

### Крок 6. Увійти на кожному пристрої
1. Відкрий свій додаток (адреса з Частини 1) → вкладка **Прогрес**.
2. У блоці *Синхронізація* введи пошту й пароль (мін. 6 символів) → **Зареєструватися**.
3. На іншому пристрої відкрий ту саму адресу → **Прогрес** → той самий email і пароль → **Увійти**.
4. Усе — тепер галочки, ваги й історія однакові скрізь і оновлюються автоматично.

## Як це працює (коротко)
- Усе працює навіть без інтернету — прогрес лягає в пам'ять браузера.
- Коли є інтернет і ти увійшов — зміни тихо вивантажуються в Supabase, а при відкритті підтягуються звідти.
- Якщо десь розійшлося — перемагає **найсвіжіша** версія за часом.
- Резервна копія файлом (Експорт/Імпорт у вкладці «Прогрес») лишилась як запасний варіант.

## Часті питання
- **Це справді безкоштовно?** Так, безкоштовних тарифів GitHub Pages і Supabase вистачає з запасом для такого додатка.
- **Чи безпечно тримати anon-ключ у config.js?** Так. Доступ до даних обмежений правилом RLS із Кроку 2: кожен бачить лише свій прогрес.
- **Забув щось?** Напиши — підкажу конкретний крок.
